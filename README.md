# EVO — Merchandising Route Planning Tool

Web tool for planners/supervisors to design, adjust, and monitor field merchandiser routes
(stores, visit frequencies, in-store tasks, schedules). See [`CLAUDE.md`](CLAUDE.md) for the
full project handbook, and [`EVO-Route-Planning-Design.md`](EVO-Route-Planning-Design.md) for
the system design.

## Prerequisites

- [.NET SDK 10](https://dotnet.microsoft.com/download) (see `docs/DECISIONS.md` — CLAUDE.md
  specifies .NET 8, but only .NET 10 was available when this scaffold was built)
- [Node.js 22+](https://nodejs.org/) and npm
- [Docker](https://www.docker.com/) (for local SQL Server + MinIO)

## Repo layout

```
backend/       .NET 10 solution — Api / Domain / Infrastructure / Tests / Seeder
panel/         React + TS web panel (Vite)
contracts/     OpenAPI contract (openapi.json) — source of truth, generated by the backend build
docs/          Architecture, decisions, API, database, roadmap, TODO
specs/         Spec-driven build: one folder per platform/feature spec
```

_(No `mobile/` yet — the React Native field-agent app is deferred; see `docs/DECISIONS.md`.)_

## Running everything

### 1. Start dev dependencies (SQL Server + MinIO)

```bash
cp .env.example .env   # first time only
docker compose -f docker-compose.dev.yml up -d
```

SQL Server: `localhost:1433` (user `sa`). MinIO: API on `localhost:9010`, console on
`localhost:9011` (remapped from the usual 9000/9001 — see `docs/DECISIONS.md`).

### 2. Backend

```bash
cd backend
dotnet run --project src/Evo.Api          # http://localhost:5076
```

Building the solution regenerates `contracts/openapi.json` automatically:

```bash
dotnet build backend/Evo.sln
```

### 3. Panel

```bash
cd panel
npm install
npm run generate-api-client   # regenerate the typed client from contracts/openapi.json
npm run dev                   # http://localhost:5173, proxies /api → backend :5076
```

## Tests

```bash
# Backend — xUnit
dotnet test backend/Evo.sln

# Panel — Vitest (unit) + Playwright (e2e smoke)
cd panel
npm test
npx playwright test
```

## Regenerating the API client

The panel never hand-writes API types. Whenever a backend endpoint changes:

```bash
dotnet build backend/Evo.sln          # refreshes contracts/openapi.json
cd panel && npm run generate-api-client   # refreshes panel/src/api/generated/ (gitignored)
```

CI fails if `contracts/openapi.json` is out of date relative to the backend code (drift check,
see `.github/workflows/ci.yml`).

## Seeding test data

`backend/src/Evo.Seeder` (Bogus-based, `--profile demo|scale --wipe`) is scaffolded as part of
spec 001; no entities exist yet to seed. Once module specs add tables:

```bash
dotnet run --project backend/src/Evo.Seeder -- --profile demo   # or: --profile scale --wipe
```
